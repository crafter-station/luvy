# Issue Tracker

This repo uses local markdown files as its issue tracker.

Issues, PRDs, and implementation tickets should be created under `.scratch/` with descriptive directory and file names. Each issue file should include enough metadata for triage at the top of the document, including title, labels, status, and creation date.

Use the triage label vocabulary documented in `docs/agents/triage-labels.md`.

When publishing a PRD, create a markdown file under `.scratch/prds/` and apply the `needs-triage` label in the file metadata so it enters the normal triage flow.
